# Learning engine

> **Deference:** This skill extends the PDCA Self-Improvement Cycle defined in CLAUDE.md. On conflict, CLAUDE.md PDCA rules govern. This skill is invoked by `/pdca-4-upgrade` as an optional post-upgrade step.

## Purpose

Extract reusable instincts from completed PDCA cycles and maintain a curated instinct registry at `.claude/pdca-archive/instincts.md`. Instincts are condensed operational lessons that inform future decision-making without requiring full PDCA re-analysis.

## When to use

- After `/pdca-4-upgrade` completes a skill modification
- When a recurring pattern is observed across 2+ PDCA cycles
- During periodic review of `.claude/pdca-archive/` to consolidate learnings
- NOT as a standalone skill -- always triggered from within a PDCA cycle

## Protocol

### Phase 1: Extract instinct from PDCA cycle

1. Read the completed PDCA cycle file at `.claude/pdca-archive/cycles/PDCA-YYYY-NNNN.md`.
2. Identify the core operational lesson -- the single sentence a developer would need to avoid repeating the mistake.
3. Classify into exactly one category:

| Category | When to use |
|----------|-------------|
| `error-pattern` | Recurring code mistake, type error, or runtime failure |
| `workflow-optimization` | Pipeline step that was slow, skipped, or misapplied |
| `quality-gate` | Missing or insufficient quality check that let a defect through |
| `architectural-principle` | Structural insight about codebase organization or boundaries |

### Phase 2: Format instinct

Each instinct follows this exact format:

```
- [PDCA-YYYY-NNNN] [category] Instinct text describing the lesson. (YYYY-MM-DD)
```

**Rules:**
- One line per instinct, no multi-line entries.
- Instinct text is imperative mood ("Always guard...", "Never use...", "Prefer...").
- Date is the date the instinct was extracted, not the incident date.
- PDCA ID links back to the source cycle for traceability.

**Examples:**
```
- [PDCA-2026-0003] [error-pattern] Always guard SemanticDependency.targetFile with undefined check before string operations. (2026-02-15)
- [PDCA-2026-0007] [workflow-optimization] Run verification loop before switching to a different task group in EIW Stage 2. (2026-02-20)
- [PDCA-2026-0012] [quality-gate] Validate Zod schema .output types in test mocks, not .input types. (2026-03-01)
- [PDCA-2026-0015] [architectural-principle] database row-level security policies must be added in the same migration that creates the table. (2026-03-10)
```

### Phase 3: Dedup check

Before adding a new instinct:

1. Read `.claude/pdca-archive/instincts.md`.
2. Check if an existing instinct covers the same lesson (same root cause, same fix).
3. If duplicate found:
   - If the new instinct is more specific/actionable, replace the old one (preserve the older PDCA ID as a secondary reference).
   - If the existing instinct is already adequate, skip insertion and log `DEDUP_SKIPPED` in the PDCA cycle file.

### Phase 4: Write instinct

1. Append the new instinct to `.claude/pdca-archive/instincts.md`.
2. Maintain the file structure:

```markdown
# PDCA instinct registry

> Auto-generated by the learning engine. Do not edit manually.
> Maximum 50 instincts. Oldest entries removed when cap is reached.

## Active instincts

- [PDCA-YYYY-NNNN] [category] Instinct text. (YYYY-MM-DD)
- [PDCA-YYYY-NNNN] [category] Instinct text. (YYYY-MM-DD)
...
```

3. If the file does not exist, create it with the header above.
4. If adding would exceed 50 instincts, remove the oldest entry (by extraction date) before appending.

### Phase 5: Confirm

Output:

```
LEARNING ENGINE: Instinct extracted
====================================
Source:    PDCA-YYYY-NNNN
Category: [category]
Instinct: [text]
Action:   ADDED / REPLACED [old-id] / DEDUP_SKIPPED
Total:    N/50 instincts
```

## Session observations (dual-channel model)

The learning engine supports two observation channels that both write to the instinct registry:

### Channel 1: PDCA incidents (existing)

- **Trigger:** Error resolution, expectation mismatch, critical feedback (via `/pdca-cycle`)
- **Prefix:** `PDCA-YYYY-NNNN`
- **Source:** Completed PDCA cycle files at `.claude/pdca-archive/cycles/`
- **Categories:** error-pattern, workflow-optimization, quality-gate, architectural-principle

### Channel 2: Session observations (new)

- **Trigger:** Manual invocation via `/learn` command or end-of-session reflection
- **Prefix:** `OBS-YYYY-NNNN`
- **Source:** Direct developer observation during a session
- **Categories:** error-pattern, workflow-optimization, quality-gate, architectural-principle, session-insight (new)
- **Format:** `- [OBS-YYYY-NNNN] [category] Observation text. (YYYY-MM-DD)`

### Channel differences

| Aspect | PDCA incidents | Session observations |
|--------|---------------|---------------------|
| Trigger | Automated after error resolution | Manual via `/learn` |
| Depth | Full 4-phase analysis | Single observation |
| Source | PDCA cycle file | Developer session |
| Prefix | `PDCA-` | `OBS-` |
| Categories | 4 standard | 4 standard + session-insight |

### Unified registry rules

- Both channels write to `.claude/pdca-archive/instincts.md`
- Combined cap: 50 entries (FIFO eviction across both channels)
- Dedup check applies across both channels (an OBS entry can replace a PDCA entry if more specific, and vice versa)
- OBS numbering: sequential starting from `OBS-YYYY-0001`, tracked alongside PDCA numbering

## FIFO cap enforcement

- Maximum 50 active instincts in the registry.
- When adding the 51st instinct, remove the entry with the oldest extraction date.
- Removed instincts remain traceable through their source PDCA cycle files in `.claude/pdca-archive/cycles/`.

## Quality gates

| Gate | Criteria | Blocking |
|------|----------|----------|
| Source PDCA exists | Cycle file at `.claude/pdca-archive/cycles/PDCA-YYYY-NNNN.md` exists | Yes |
| Single category | Exactly one category assigned | Yes |
| Format valid | Matches `- [PDCA-YYYY-NNNN] [category] text (YYYY-MM-DD)` | Yes |
| Dedup checked | Existing registry scanned for duplicates | Yes |
| Cap enforced | Registry has <= 50 entries after write | Yes |

## Common pitfalls

- **Instincts that are too vague** -- "Be careful with types" is useless. "Always check `.output` vs `.input` when using Zod schemas with `.default()`" is actionable.
- **Multiple lessons per instinct** -- each instinct captures exactly one lesson. If a PDCA cycle yields two insights, create two instincts.
- **Manual edits to instincts.md** -- the file header says "do not edit manually" for a reason. All changes go through this protocol to maintain traceability.
- **Skipping dedup** -- duplicate instincts dilute the registry. Always scan before adding.
